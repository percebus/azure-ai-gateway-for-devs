## Summary

Explore the reverse integration: instead of configuring the linkage between Foundry and APIM using the APIM wizards, let's try it from the other side, and have Foundry create the linkage to APIM for you and inspect what it generates.

## Goals

- Comparing different wizard-generated APIM configuration
- Understanding what Foundry auto-provisions (APIs, backends, products, subscriptions)
- Discovering how Foundry subscription keys map to APIM subscription keys

## Objective

This is a bonus exercise and it's not part of the main tutorial.

We'll create another linkage between APIM and Foundry. But this time, we'll do it from Foundry instead, and compare against what we've built by hand.

## Skills you will learn

- Adding an AI Gateway from the Foundry portal
- Inspecting auto-generated APIM APIs, backends, and products
- Comparing Managed Identity resource IDs across manual vs wizard setups
- Understanding Foundry-to-APIM subscription key mapping
- Evaluating trade-offs: clone-and-cleanup vs build-from-scratch
